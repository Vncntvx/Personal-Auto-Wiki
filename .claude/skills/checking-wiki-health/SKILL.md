# Wiki 健康检查

定期或按需检查 Wiki 知识库的完整性和一致性。

## 触发条件

- 用户请求"健康检查"、"检查 Wiki 状态"、"诊断"
- 摄取操作后验证结果
- 定期维护

## 检查项目

### 1. 孤立页面检测

**定义**：无入链的页面（未被任何 Wiki 页面引用）

**检测方法**：
1. 遍历所有 Wiki 页面
2. 提取所有 `[[wiki-links]]` 引用
3. 识别未被引用的页面

**处理建议**：
- 如有价值，在其他页面添加引用
- 如无价值，考虑删除或合并

### 2. 未索引页面检测

**定义**：存在于文件系统但未在 `Wiki/Index.md` 中列出的页面

**检测方法**：
1. 列出 `Wiki/Concepts/*.md`、`Wiki/Entities/*.md`、`Wiki/Sources/*.md`
2. 与 `Index.md` 中的条目对比

**处理建议**：更新 Index.md 添加缺失条目

### 3. 概念缺口检测

**定义**：在页面中引用但目标页面不存在

**检测方法**：
1. 扫描所有页面中的 `[[Concepts/xxx]]`、`[[Entities/xxx]]` 链接
2. 检查目标文件是否存在

**处理建议**：创建缺失的概念/实体页面

### 4. 矛盾标注检测

**定义**：存在 `> [!warning] 矛盾标注` 的页面

**检测方法**：搜索所有页面中的矛盾标注

**处理建议**：人工审核矛盾内容

### 5. 未同步文件检测

**定义**：Resources 目录中存在但 Wiki/Sources 中无对应页面，或内容已变更

**检测方法**：调用 `detecting-resources-sync` Skill

**处理建议**：调用 `ingesting-resources` 摄取

## 输出格式

```markdown
# Wiki 健康检查报告

日期：YYYY-MM-DD HH:MM

## 统计

| 类型 | 数量 |
|------|------|
| 概念页面 | 54 |
| 实体页面 | 34 |
| 来源页面 | 45 |
| 产出页面 | 0 |
| **总计** | **133** |

## 问题列表

### 孤立页面 (N)

- [[Concepts/xxx]] — 无入链
- [[Entities/yyy]] — 无入链

### 未索引页面 (N)

- Wiki/Concepts/zzz.md — 未在 Index.md 中列出

### 概念缺口 (N)

- [[Concepts/aaa]] — 被 [[Sources/bbb]] 引用但不存在

### 矛盾标注 (N)

- [[Concepts/ccc]] 与 [[Sources/ddd]] 存在矛盾

### 未同步文件 (N)

- `Academic/NewTopic.md` — 新增待摄取
- `Academic/Theory/韧性.md` — 内容已变更

## 建议操作

1. [具体建议]
2. [具体建议]
```

## 检查优先级

| 优先级 | 检查项 | 原因 |
|--------|--------|------|
| 高 | 未同步文件 | 影响知识库完整性 |
| 高 | 概念缺口 | 影响引用解析 |
| 中 | 矛盾标注 | 影响知识一致性 |
| 低 | 孤立页面 | 不影响核心功能 |
| 低 | 未索引页面 | 仅影响目录浏览 |

## 相关 Skill

- 调用：`detecting-resources-sync`（检测未同步文件）
- 后续：`ingesting-resources`（处理未同步文件）
