# 健康检查清单

## 检查流程

### 1. 统计信息

- [ ] 统计 Concepts 数量
- [ ] 统计 Entities 数量
- [ ] 统计 Sources 数量
- [ ] 统计 Outputs 数量

### 2. 孤立页面检测

- [ ] 扫描所有页面
- [ ] 提取所有 wiki-links
- [ ] 识别未被引用的页面
- [ ] 评估孤立页面价值

### 3. 未索引页面检测

- [ ] 列出 Concepts 目录文件
- [ ] 列出 Entities 目录文件
- [ ] 列出 Sources 目录文件
- [ ] 与 Index.md 对比

### 4. 概念缺口检测

- [ ] 扫描所有 `[[Concepts/xxx]]` 链接
- [ ] 扫描所有 `[[Entities/xxx]]` 链接
- [ ] 检查目标文件是否存在

### 5. 矛盾标注检测

- [ ] 搜索 `> [!warning] 矛盾标注`
- [ ] 收集所有矛盾位置
- [ ] 评估矛盾严重性

### 6. 未同步文件检测

- [ ] 调用 detecting-resources-sync
- [ ] 识别新增文件
- [ ] 识别变更文件
- [ ] 识别删除文件

## 问题优先级

| 优先级 | 问题类型 | 处理建议 |
|--------|----------|----------|
| P0 | 未同步文件 | 立即摄取 |
| P1 | 概念缺口 | 创建缺失页面 |
| P1 | 矛盾标注 | 人工审核解决 |
| P2 | 未索引页面 | 更新 Index.md |
| P3 | 孤立页面 | 评估后处理 |

## 定期检查建议

- **每日**：检查未同步文件（如有摄取活动）
- **每周**：完整健康检查
- **每月**：清理孤立页面、更新索引
